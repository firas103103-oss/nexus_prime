# القسم الأول: الوثيقة التنفيذية الشاملة

## 1. الرؤية العامة

بناء **نظام إيكولوجي رقمي موحد** تحت شركة قابضة (MRF)، يجمع أربع وحدات عمل رئيسية + مبادرات عامة، كلها مرتبطة بقاعدة بيانات واحدة، على سيرفر محلي سيادي، مع تسلسل واضح للمستخدمين (زائر → مسجل → مشترك) ونظام تسعير واشتراكات.

---

## 2. المستوى الأول: اللانينق الرئيسي (mrf103.com)

### 2.1 الوظيفة
- نقطة الدخول الأولى للقابضة والبراند الرئيسي.
- تعريف، رؤية، تخصصات، مبادرات، إنجازات، زخم عام.
- بعد الاستعراض → تفرع واضح للمشاريع والمنتجات والخدمات.

### 2.2 متطلبات التصميم
| البند | التفاصيل |
|-------|----------|
| الفخامة | تصميم استثنائي — ليست عادية |
| الحيوية | رسومات، حركة، حياة — الموقع "يتنفس" |
| الاحترافية | واجهة متقدمة جداً |
| الوضوح | الرؤية السيادية بلغة مبسطة للجميع |
| التفرع | روابط واضحة للفروع الأربعة + المبادرات |

### 2.3 المحتوى (ترتيب العرض)
1. الرؤية السيادية  
2. التخصصات  
3. المبادرات  
4. الإنجازات  
5. الزخم العام  
6. التفرع: أربع بطاقات رئيسية + قسم المبادرات العامة + قسم "متفرقات"

---

## 5. التسلسل الهرمي للمستخدمين

### 5.1 محور المالك (Super Admin)
- صاحب ومالك ومشغّل النظام بالكامل.
- صلاحيات كاملة: مراقبة، تحكم، تخصيص، إدارة، إدخال، تعديل، حذف.
- يشمل: كل مكون، كل كوكب، كل فاتورة، كل كود.

### 5.2 تجربة المستخدم (User Experience)

| الطبقة | الوصف | الوصول |
|--------|-------|--------|
| 1. زائر عابر | متصفح فقط | محتوى عام، أغلب الخدمات مغلقة |
| 2. مستخدم مسجل | قام بتسجيل الدخول | غير مشترك — أغلب الأمور مغلقة (اشتراك/تفعيل/دفع) |
| 3. مستخدم مشترك | مسجل + مشترك | ربط التسجيل بالاشتراك، ترقية الاستخدام |
| 4. مشترك متعدد | مشترك بأكثر من خدمة | ربط متعدد، صلاحيات حسب الاشتراكات |

### 5.3 القفل والفتح
- الزائر: محتوى عام فقط.
- المسجل غير المشترك: واجهات مرئية، قفل الخدمات (رسالة: اشتراك/تفعيل/دفع).
- المشترك: فتح حسب نوع الاشتراك.

# القسم الثاني: الخطة التنفيذية المباشرة

## المرحلة أ — اللانينق (الأولوية العليا)

1. **ترقية landing-pages/index.html**
   - إعادة تصميم مع فخامة واستثنائية أعلى.
   - إضافة رسومات وحيوية (animations, illustrations).
   - كتابة نصوص الرؤية السيادية بلغة مبسطة.

2. **إضافة أقسام المحتوى**
   - قسم الرؤية السيادية.
   - قسم التخصصات.
   - قسم المبادرات.
   - قسم الإنجازات.
   - قسم الزخم العام.

3. **قسم التفرع**
   - أربع بطاقات رئيسية: دار النشر، النظام التشغيلي، اكس بيو، الرابع.
   - قسم المبادرات العامة (القارئ القرآني، تقويم نيكسس، نيكسس كور داتا).
   - قسم متفرقات.
   - كل بطاقة → رابط واضح للمنصة.

4. **النشر**
   - التأكد من النشر إلى `/var/www/nexus-landing`.
   - التحقق من سكربت النشر: `scripts/deploy_frontends.sh`.

---

## المرحلة ب — التفرع والربط

1. **تحديث الروابط في اللانينق**
   - دار النشر → publisher.mrf103.com
   - النظام التشغيلي → platform.mrf103.com أو sovereign.mrf103.com
   - اكس بيو → xbio.mrf103.com
   - الرابع → boardroom.mrf103.com (Boardroom ضمن الرابع)

2. **تنظيم Boardroom**
   - نقل Boardroom conceptually تحت "الرابع" في التنظيم والروابط.

---

## المرحلة ج — التوثيق

1. **تحديث docs/PROJECTS_MAP.md**
   - إضافة قسم "الفروع الأربعة" ليعكس الهيكلة الجديدة.

2. **تحديث docs/LANDING_AND_HIERARCHY_VISION.md**
   - إضافة المبادرات العامة إن لزم.
   - تحديث الروابط للرابع (boardroom).

# القسم الثالث: البرومت للإيجنت

**انسخ النص التالي وألصقه للإيجنت لتحقيق أفضل نتيجة:**

---

```
المهمة: ترقية لانينق mrf103.com وإنشاء الهيكلة التفرعية حسب الوثيقة docs/EXECUTION_PLAN_AND_AGENT_PROMPT.md

السياق:
- المشروع: NEXUS PRIME / MRF103
- الملف المرجعي الرئيسي: docs/EXECUTION_PLAN_AND_AGENT_PROMPT.md
- الوثيقة التفصيلية: docs/LANDING_AND_HIERARCHY_VISION.md
- اللانينق الحالي: landing-pages/index.html
- مسار النشر: /var/www/nexus-landing
- سكربت النشر: scripts/deploy_frontends.sh

الهدف: واجهة فخمة احترافية تعرض الرؤية السيادية بلغة مبسطة، مع تفرع واضح لأربعة فروع رئيسية + مبادرات عامة.

المتطلبات:
1. التصميم: فخامة، حيوية (رسومات/حركة)، احترافية، استثنائية — الموقع "يتنفس".
2. المحتوى: أقسام — الرؤية السيادية، التخصصات، المبادرات، الإنجازات، الزخم العام.
3. التفرع: أربع بطاقات رئيسية:
   - دار النشر + الظل السابع → publisher.mrf103.com
   - النظام التشغيلي الذكي الحيو السيادي → platform.mrf103.com / sovereign.mrf103.com
   - اكس بيو (AIoT) → xbio.mrf103.com
   - الرابع (مكاتب، اجتماعات، استنساخ، خلق) → boardroom.mrf103.com
4. المبادرات: قسم للمبادرات العامة (القارئ القرآني الكوني، تقويم نيكسس، نيكسس كور داتا، متفرقات).
5. اللغة: عربي RTL، لغة مبسطة للجميع.
6. التوافق: متصفحات حديثة + موبايل.
7. الألوان: يمكن الاحتفاظ بالألوان الحالية (cyan, magenta, purple) أو ترقيتها لفخامة أعلى.

المراجع المطلوبة (اقرأها قبل التنفيذ):
- docs/EXECUTION_PLAN_AND_AGENT_PROMPT.md
- docs/LANDING_AND_HIERARCHY_VISION.md
- landing-pages/index.html
- docs/PROJECTS_MAP.md
- docs/nginx-frontend-routes.md

القيود:
- عدم المساس بالـ API والخدمات الخلفية.
- عدم تغيير بنية nginx أو المنافذ.
- اللانينق HTML/CSS/JS ثابت (أو React إن كان مشروع اللانينق يستخدمه — تحقق أولاً).

المخرجات المتوقعة:
1. تعديل/إنشاء landing-pages/ حسب الرؤية والخطة.
2. تحديث سكربت النشر إن لزم.
3. نشر اللانينق إلى /var/www/nexus-landing.
4. ملاحظات على النصوص/الصور التي تحتاج إدخال يدوي من المالك.

الترتيب: ابدأ بالمرحلة أ (اللانينق) ثم ب (التفرع) ثم ج (التوثيق).
```
